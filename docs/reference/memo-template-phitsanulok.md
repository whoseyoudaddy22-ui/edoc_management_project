# Reference: Memo Template — บันทึกข้อความ (ตัวอย่างจริง)

> ไฟล์นี้เป็นเอกสารอ้างอิงเพิ่มเติม ใช้คู่กับ skill `official-document-template`
> และ Module 5 (สร้างเอกสาร) / Module 8 (พิมพ์เอกสาร/PDF)
> ที่มา: ตัวอย่างเอกสารจริงจากองค์การบริหารส่วนจังหวัดพิษณุโลก (บันทึกข้อความ)

## วัตถุประสงค์

เอกสารตัวอย่างจริงที่ได้รับมามีโครงสร้างเฉพาะของ "บันทึกข้อความ" ซึ่งต่างจาก "หนังสือภายนอก" เล็กน้อย (ไม่มีคำลงท้าย "ขอแสดงความนับถือ" แบบหนังสือภายนอก แต่ใช้ "จึงเรียนมาเพื่อทราบ...") ให้ใช้ไฟล์นี้เป็นต้นแบบเวลาสร้าง/แก้ไขฟอร์ม "สร้างเอกสาร" และ template "พิมพ์เอกสาร" สำหรับประเภทเอกสารนี้โดยเฉพาะ

มีไฟล์ต้นแบบ HTML (`memo-form.html`) แนบมาด้วย เป็น static prototype ที่สร้างไว้ก่อนแล้ว ให้ใช้เป็นแนวทางโครงสร้าง CSS Grid/Flexbox และ field layout เมื่อนำไปประกอบเป็น React component จริงในโปรเจกต์

## โครงสร้างฟิลด์ของ "บันทึกข้อความ" (แตกต่างจากหนังสือภายนอกที่มีอยู่)

| ฟิลด์ในฟอร์ม | Label ภาษาไทย | ผูกกับ field ใน `Document` model | หมายเหตุ |
|---|---|---|---|
| ส่วนราชการ | ส่วนราชการ | `departmentName` (ชื่อเต็ม + เบอร์โทร รวมในบรรทัดเดียว) | ต่างจาก `departmentCode` ที่ใช้ในเลขที่เอกสาร — ฟิลด์นี้คือชื่อเต็มแสดงในเอกสาร |
| ที่ | เลขที่หนังสือ | `documentNumber` | ดึงจาก skill `document-numbering` โดยอัตโนมัติ ห้ามให้ผู้ใช้พิมพ์เอง |
| วันที่ | วันที่หนังสือ | `createdAt` (format เป็น "เดือน ปี พ.ศ." แบบไทย ไม่ต้องมีวันที่ถ้าเป็น บันทึกข้อความ ต่างจากหนังสือภายนอกที่ระบุวันที่เต็ม) | สังเกตจากตัวอย่างจริง: ใช้แค่ "พฤษภาคม 2569" ไม่มีเลขวันที่ — ต้องรองรับ format แบบนี้สำหรับประเภท "บันทึกข้อความ" โดยเฉพาะ |
| เรื่อง | เรื่อง | `title` | เหมือน Module 5 เดิม |
| เรียน | ผู้รับ | `recipient` | เหมือน Module 5 เดิม |
| เนื้อความ | เนื้อหาเอกสาร | `content` | รองรับ text แบบมี placeholder จุดประ (เช่น `.../2569 ลงวันที่........`) ที่ผู้ใช้กรอกเติมเองได้ — ต้องคิดว่าจะให้เป็น rich text ธรรมดา หรือมี field ย่อยสำหรับเลขที่คำสั่ง/วันที่คำสั่งอ้างอิงแยกต่างหาก |
| จึงเรียนมาเพื่อทราบฯ | ข้อความปิดท้าย | `closingText` (ใหม่ — ต้องเพิ่มใน schema ถ้ายังไม่มี) | บันทึกข้อความใช้ "จึงเรียนมาเพื่อทราบ และดำเนินการตามอำนาจและหน้าที่ต่อไป" ต่างจากหนังสือภายนอกที่มักใช้ "ขอแสดงความนับถือ" — ควรเป็น dropdown เลือกข้อความมาตรฐาน ไม่ใช่ให้พิมพ์เองทุกครั้ง เพื่อความสม่ำเสมอ |
| ลายเซ็น/ตำแหน่ง | ผู้ลงนาม | `signerName`, `signerPosition` | เหมือน Module 5 เดิม |

## สิ่งที่ต้องตรวจสอบ/แก้ไขก่อนนำไปใช้จริง (สำคัญ)

1. **ฟิลด์ `closingText` อาจยังไม่มีใน Prisma schema ปัจจุบัน** — ถ้ายังไม่มี ต้องเพิ่มเป็น enum หรือ text field ที่มีค่าเริ่มต้นตาม `documentTypeCode` (เช่น "บันทึกข้อความ" default เป็น "จึงเรียนมาเพื่อทราบ...", "หนังสือภายนอก" default เป็น "ขอแสดงความนับถือ")
2. **Format วันที่ที่ต่างกันตามประเภทเอกสาร** — บันทึกข้อความในตัวอย่างจริงใช้แค่ "เดือน ปี" ไม่มีวันที่ ต้องเช็คว่า logic การ format วันที่ในระบบรองรับกรณีนี้หรือยัง หรือจะบังคับให้มีวันที่เต็มทุกประเภทเอกสารเพื่อความสม่ำเสมอ (ต้องตัดสินใจร่วมกับผู้ใช้งานจริงของระบบ)
3. **ตราครุฑ** — ไฟล์ต้นแบบ `memo-form.html` ใส่เป็น placeholder ("ตราครุฑ") ไว้ก่อน ต้องใช้ไฟล์ภาพตราครุฑที่ถูกต้องของหน่วยงานจริงแทนที่ ก่อนใช้งานจริง (ห้ามสร้างขึ้นเองแบบไม่เป็นทางการ)
4. **ฟอนต์** — ต้นแบบใช้ Google Fonts "Sarabun" เป็น fallback เพราะ TH Sarabun New/PSK ไม่ใช่ web-safe font ตามที่ระบุไว้ใน skill `official-document-template` อยู่แล้ว — ต้องฝัง `@font-face` จริงตาม skill นั้นเมื่อนำไปใช้ใน production ไม่ใช่พึ่ง Google Fonts CDN ตลอดไป (เพราะระบบอาจต้องใช้งานแบบ offline/intranet)

## งานที่ต้องให้ Claude Code ทำ

1. เทียบ `memo-form.html` กับ component ปัจจุบันของ Module 5 (ฟอร์มสร้างเอกสาร) และ Module 8 (template พิมพ์เอกสาร)
2. เพิ่ม field `closingText` เข้า Prisma schema ถ้ายังไม่มี พร้อม migration
3. ปรับ dropdown "ประเภทเอกสาร" ให้เมื่อเลือก "บันทึกข้อความ" แล้ว auto-fill ค่าเริ่มต้นของ `closingText` และปรับ format วันที่ให้ตรงกับตัวอย่างจริง
4. นำโครงสร้าง CSS Grid/Flexbox จาก `memo-form.html` ไปปรับใช้กับ shared component เอกสาร (`src/components/shared/document-template.tsx`) ตามที่ระบุไว้ใน skill `official-document-template` เรื่อง Preview/PDF parity — ห้ามสร้าง component แยกต่างหากสำหรับประเภทเอกสารนี้โดยเฉพาะ ให้ใช้ conditional rendering ภายใน component เดียวกัน
5. เพิ่ม placeholder ตราครุฑแบบเดียวกับต้นแบบ พร้อม comment ในโค้ดว่าต้องแทนที่ด้วยไฟล์จริงก่อน deploy
